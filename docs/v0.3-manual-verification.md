# troxy v0.3 — `troxy start` 수동 검증 체크리스트

자동 테스트로 잡히지 않는 UX / 실제 프록시 연동 부분을 사람이 한 번 돌려보기 위한 가이드.
릴리즈 전, 또는 TUI 관련 큰 변경이 들어갈 때마다 수행.

## 사전 준비

```bash
# 1) 빌드 (editable)
uv sync

# 2) 깨끗한 DB로 시작
export TROXY_DB=/tmp/troxy-verify.db
rm -f "$TROXY_DB"

# 3) 브라우저/단말에서 127.0.0.1:18090 프록시 설정 (또는 실기기는 Mac IP:18090)
# 4) mitm CA 설치는 https://mitm.it 안내에 따른다
```

## 실행

```bash
uv run troxy start -p 18090
```

## 체크리스트

### 기본 화면

- [ ] TUI가 뜨고 상단에 `{DB path} · 0 flows` 헤더 표시
- [ ] 하단에 `📡 {my-ip}` 표기가 나온다 (30초마다 갱신)
- [ ] mitmdump 프로세스가 백그라운드에서 동작 (`lsof -i :18090`)

### 실시간 flow 수집

- [ ] 브라우저에서 HTTP(S) 요청 발생 → 0.5초 이내 표 갱신
- [ ] 헤더의 flow 수가 증가
- [ ] status code별 색 / 아이콘이 올바름 (2xx ✓ 초록 · 3xx — 파랑 · 4xx ⚠ 노랑 · 5xx 🔥 빨강)

### 네비게이션

- [ ] ↑ / ↓ 로 flow 탐색, 커서 위치가 시각적으로 유지됨
- [ ] Enter → DetailScreen 진입
- [ ] DetailScreen에서 Request / Response 세로 분할 표시, 디폴트 포커스는 Response
- [ ] Tab / Shift+Tab → Request ↔ Response 포커스 전환 (focused-pane 보더 표시)
- [ ] Esc → ListScreen 복귀 (리스트 상태·커서 유지)

### 복사

- [ ] `y` → copy modal 뜨고 1 URL / 2 Request / 3 Response / 4 Body / 5 curl / 6 httpie 표시
- [ ] 1 선택 → URL이 클립보드에 복사되고 ✓ 토스트
- [ ] 2 선택 → 전체 Request (method + URL + headers + body) 복사
- [ ] 3 선택 → 전체 Response (status + headers + body) 복사
- [ ] 4 선택 → response body만 복사
- [ ] 5 선택 → `curl …` 명령어 복사 (터미널 붙여넣기로 실행 확인)
- [ ] 6 선택 → `http …` (HTTPie) 명령어 복사
- [ ] `Y` (Shift+Y) → 현재 포커스 패널(Request/Response) 즉시 복사
- [ ] `u` → URL 즉시 복사
- [ ] `c` → curl 즉시 복사
- [ ] 토스트 메시지 `✓ copied: {what} ({size})` 표시 후 자동 사라짐

### 필터 (ListScreen)

- [ ] `f` → 상단 FilterInput 표시, 포커스
- [ ] `host:example.com` 입력 후 Enter → 일치하는 flow만 표시
- [ ] `status:4xx` 입력 후 Enter → 4xx 만 표시
- [ ] `method:POST` 입력 후 Enter → POST 만 표시
- [ ] 여러 조건 AND 결합 확인 (`host:example.com status:500`)
- [ ] 필터 적용 중엔 새 flow가 화면에 추가되지 않음 (폴링이 스킵)
- [ ] Esc → 필터 해제, 전체 목록 + 실시간 폴링 재개

### Mock (M-대문자 / m-소문자)

- [ ] 소문자 `m` → MockDialog prefill (host/path/method/status/body)
- [ ] path에 glob 제안이 보임 (예: `/api/v2/users/:id/...`)
- [ ] 저장 → ✓ mock saved: {name} (enabled) 토스트
- [ ] 같은 이름으로 저장 → 중복 에러 메시지
- [ ] 대문자 `M` → MockListScreen 진입
- [ ] Space → 선택 mock toggle (enabled/disabled 상태 반영)
- [ ] Enter → 편집 모드 (MockDialog prefill)
- [ ] `d` → 삭제 confirm → y → ✓ mock deleted 토스트

### Clear

- [ ] `x` → confirm dialog "전체 N개 flow를 삭제합니다. 확인? [y/N]"
- [ ] `y` → 전체 flow 삭제, ✓ {N} flows cleared 토스트, 헤더 0으로
- [ ] `n` / Esc → 취소, 목록 그대로

### Intercept placeholder

- [ ] `i` → "intercept live edit — coming in v0.4" 토스트
- [ ] 실제 intercept UI는 열리지 않음 (플레이스홀더)

### 종료

- [ ] `q` → TUI 종료
- [ ] mitmdump 프로세스도 함께 종료 (`lsof -i :18090` 비어 있음)
- [ ] 종료 후 SQLite DB 파일은 남아 있음 (`$TROXY_DB`)

## 알려진 제약 (문서화만, 검증 대상 아님)

- v0.3에서는 `action_replay` (`r` 키) / intercept live editing은 의도적으로 미구현
- MockDialog / MockListScreen에서의 실시간 hit_count 갱신은 MockListScreen 재진입 시 반영
- `my-ip` 표시는 LAN 기준, 공용 네트워크/VPN 환경에선 부정확할 수 있음
