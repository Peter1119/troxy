# troxy v0.3 — Start TUI + Mock Dialog Design Spec

> `troxy start` 하나로 프록시 실행 + 실시간 인스펙션 + 1-click mock.

## Goal

v0.3에서 `troxy start`를 Textual TUI로 교체한다.
한 명령으로 프록시가 켜지고 인스펙터가 뜬다. Charles처럼.
기존 one-shot CLI(`troxy flows`, `troxy flow 42`)는 건드리지 않는다.

## `troxy start` 동작 변경

**Before (v0.2):** `troxy start` → mitmproxy TUI 실행 (addon 포함)
**After (v0.3):** `troxy start` → mitmdump(headless) 백그라운드 + troxy TUI 전면

- `mitmdump -s addon.py -p 8080` 을 subprocess로 실행 (TUI 없는 headless 프록시)
- troxy Textual TUI가 전면에서 DB polling으로 flow 표시
- `q`로 TUI 종료 시 mitmdump subprocess도 함께 종료
- `--port`, `--mode` 옵션 기존대로 유지

## North Star

**B — Charles/Proxyman을 대체하는 CLI-first 프록시 인스펙터**
A(AI-ready MCP)와 C(mobile dev 집중)를 포함하되, 터미널 UX가 제품의 중심.

## Success Metrics

- `troxy start` 실행 → 첫 flow 도달까지 3초 이내 (설치된 상태 기준)
- 화면 처음 본 사람이 키바인딩 없이 필터/mock 만들기 성공
- 기존 v0.2 테스트 전부 green + watch/mock 신규 테스트
- Textual import 시간 < 500ms (lazy import)

## In-Scope

1. `troxy start` — mitmdump(headless) 백그라운드 + Textual TUI 전면. q 종료 시 mitmdump도 종료.

## Architecture

### 레이어 확장

```
troxy start
    │ spawns subprocess
    ▼
mitmdump -s addon.py -p 8080  (headless, no TUI)
    │
    troxy addon (flow → SQLite)
    │ writes
    ▼
SQLite (flows.db)
    │ reads
┌───┼────┬────┐
▼   ▼    ▼    ▼
CLI MCP  TUI  (new, foreground)
              q → SIGTERM mitmdump
```

`troxy start`가 mitmdump를 subprocess로 띄우고, Textual TUI가 전면에서 동작.
TUI 종료(q) 시 mitmdump에 SIGTERM 전송하여 함께 종료.

`src/troxy/tui/` 디렉토리 신설. core만 의존, mitmproxy import 금지.

### Dependency Rule (확장)

`addon → core ← cli, mcp, tui`

### 신규 파일 구조

```
src/troxy/tui/
├── __init__.py
├── app.py              # TroxyStartApp (textual.App)
├── proxy.py            # ProxyManager (mitmdump subprocess 관리, SIGTERM)
├── list_screen.py      # ListScreen (기본 화면, FlowTable)
├── detail_screen.py    # DetailScreen (Enter로 push)
├── mock_dialog.py      # MockDialog modal
├── mock_list.py        # MockListScreen (M 키)
├── widgets.py          # FilterInput, ConfirmDialog, CopyModal, Toast
├── theme.py            # 컬러 팔레트, 스타일 상수
├── copy.py             # 모든 사용자 노출 텍스트 (힌트, 토스트, 라벨)
└── network.py          # NetworkMonitor (my ip 탐지)
```

### 의존성 추가

- `textual` (rich 기반, 이미 rich 사용 중)
- 기본 설치에 포함할지 `pip install troxy[tui]` optional로 할지는 eng review에서 결정

## UI Design

### List View (기본 화면)

```
┌─ troxy start ──────────────── ~/.troxy/flows.db · 1,247 flows ────────┐
│                                                                        │
│  #   TIME     METHOD  HOST              PATH                 STATUS   │
│ ▶42  09:55:13  GET    api.example.com  /api/users/.../r…    401 ⚠   │
│  43  09:55:14  POST   api.example.com  /api/users           200 ✓   │
│  44  09:55:15  GET    api.example.com  /api/movies/123      304     │
│  45  09:55:16  GET    api.github.com    /repos/foo           200 ✓   │
│  46  09:55:17  PUT    api.example.com  /api/ratings         500 🔥  │
│  47  09:55:18  GET    cdn.example.com   /assets/img.png      200 ✓   │
│  48  09:55:19  POST   api.example.com  /api/login           401 ⚠   │
│  49  09:55:20  GET    api.example.com  /api/me              200 ✓   │
│  50  09:55:21  DELETE api.example.com  /api/ratings/42      204 ✓   │
│  ...                                                                   │
│                                                                        │
│  (flow 목록이 화면 대부분을 차지)                                        │
│                                                                        │
├────────────────────────────────────────────────────────────────────────┤
│ ↑↓ browse · ⏎ detail · f filter · m mock · M mocks · x clear · q quit │
│ 📡 192.168.0.56  ·  💡 ask Claude via MCP                               │
└────────────────────────────────────────────────────────────────────────┘
```

#### 핵심 원칙

- **힌트 바 상시 표시** — 하단에 available action 전부 노출. 외울 필요 없음.
- **선택된 행이 또렷이** — 좌측 `▶`, 배경색 하이라이트.
- **색상 시맨틱** — 2xx green, 3xx blue, 4xx yellow, 5xx red. 메서드도 동일 규칙.
- **정렬은 항상 time desc 고정** (newest first). 사용자 제어 없음.
- **my ip 하단 상시** — 기기에서 프록시 꽂을 때 쓸 IP.
- **Claude hint는 MCP 등록된 경우에만** — 한 줄 배너.

#### Filter (`f` 키)

- `f` 누르면 **상단에 filter 입력란 나타남**
- 텍스트 입력 후 Enter → 반영, 입력란 사라짐
- Esc → 취소, 입력란 사라짐
- 필터 활성 시 하단에 요약: `🔍 host:api.* 4xx (3 active) · Esc to reset`
- 필터 비활성 시 그 줄은 사라짐
- 문법: `host:X`, `status:4xx`, `method:POST`, `path:/api/*`, 텍스트 전문 검색

#### Clear (`x` 키)

- `x` 누르면 confirm modal: "전체 N개 flow를 삭제합니다. 확인? [y/N]"
- 항상 전체 삭제 (필터 무관)
- 확인 시 `core.query.delete_all_flows()` 호출, 목록 즉시 갱신

### Detail View (Enter로 진입)

```
┌─ Flow #42 · GET /api/users/17ov.../ratings · 401 · 30ms ────────────┐
│ URL: https://api.example.com/api/users/17ovVJlDr1vzy/ratings/2026/4 │
├─ Request ▌(focused) ─────────────────────────────────────────────────┤
│ Authorization: Bearer eyJ...                                          │
│ Accept: application/json                                              │
│ User-Agent: Example/1.0 (iOS)                                          │
│                                                                       │
│ (body 없음)                                                           │
│                                                                       │
├─ Response (401) · 152 bytes ─────────────────────────────────────────┤
│ Content-Type: application/json                                        │
│                                                                       │
│ {                                                                     │
│   "error": "unauthorized",                                            │
│   "message": "Invalid or expired token"                               │
│ }                                                                     │
│                                                                       │
├───────────────────────────────────────────────────────────────────────┤
│ Tab switch · ↑↓ scroll · y copy · u copy url · m mock · c curl · Esc  │
└───────────────────────────────────────────────────────────────────────┘
```

#### 포커싱

- **Tab / Shift+Tab** → Request 패널 ↔ Response 패널 전환
- 포커스된 패널: 좌측 `▌` 표시 + 헤더 하이라이트
- 포커스 패널 안에서 **↑↓** 스크롤
- 기본 포커스 = Response (디버깅 시 응답이 주로 궁금)

#### 레이아웃

- Request/Response **세로 분할 (vertical)**
- **1:2 비율 고정** (Response가 2배). 스크롤로 해결.

#### Copy (`y` 키 → 모달)

```
┌─ Copy ──────────────────┐
│  [1] URL                 │
│  [2] Request (full)      │
│  [3] Response (full)     │
│  [4] Response body only  │
│  [5] as curl             │
│  [6] as HTTPie           │
└──────────────────────────┘
```

- 숫자 키 또는 ↑↓+Enter로 선택
- 클립보드: macOS `pbcopy`, Linux `xclip`/`wl-copy`
- 선택 후 토스트: `✓ copied: Request (1.2 KB)` 하단 2초 표시

#### 단축 copy (파워유저)

- 포커스가 Request일 때 `Y` (대문자) → Request 즉시 copy
- 포커스가 Response일 때 `Y` → Response 즉시 copy
- `u` 어디서나 → URL copy

### Mock Dialog (`m` 키)

```
┌─ Mock from Flow #42 ────────────────────────────────────────────┐
│                                                                  │
│  Name:        [users-401_____________________] (optional)        │
│                                                                  │
│  Match:       GET  api.example.com  /api/users/*/ratings/*/*   │
│               [x] path exact    [ ] use glob (suggested)         │
│                                                                  │
│  Response:                                                       │
│    Status:    [401▼]                                             │
│    Headers:   Content-Type: application/json                     │
│               [+ add header]                                     │
│    Body:      ┌─────────────────────────────────────────────┐   │
│               │ {                                            │   │
│               │   "error": "unauthorized",                   │   │
│               │   "message": "Invalid or expired token"     │   │
│               │ }                                            │   │
│               └─────────────────────────────────────────────┘   │
│                                                                  │
│  [x] Enable immediately                                          │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│  Tab next field · ⏎ save · Esc cancel                           │
└──────────────────────────────────────────────────────────────────┘
```

#### 동작

1. **Prefill** — 선택된 flow의 method/host/path/status/headers/body 전부 채움. 바꾸고 싶은 것만 수정.
2. **Path glob 자동 제안** — path에 숫자/UUID → glob 제안 (`/api/users/17ov.../` → `/api/users/*/`). toggle으로 exact/glob 선택.
3. **Status 드롭다운** — 200/201/204/400/401/403/404/500/502/503.
4. **Body 편집** — TextArea. 작은 body 인라인, 큰 body `z`키로 확장.
5. **Name 자동 제안** — `{path-last-segment}-{status}` (e.g. `ratings-401`).
6. **Save** → `core.mock.add()`, 토스트 `✓ mock saved: users-401 (enabled)`.

### Mock List (`M` 키)

```
┌─ Mocks ─────────────────────────────────────────────────────────┐
│ ON  #  NAME                MATCH                     STATUS  HIT │
│ ●   1  users-401          GET api.*/users/*/r…      401    12  │
│ ●   2  login-success      POST api.*/login           200     3  │
│ ○   3  server-down        GET api.*                  500     0  │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│ ↑↓ browse · Space toggle · ⏎ edit · d delete · Esc back         │
└──────────────────────────────────────────────────────────────────┘
```

- `●` enabled, `○` disabled
- **HIT** 컬럼 — mock 매칭 횟수
- **Space** 토글, **d** 삭제 (confirm), **Enter** 편집 (MockDialog 재사용)

## Data Changes

### Schema Migration (v0.3)

```sql
ALTER TABLE mock_rules ADD COLUMN hit_count INTEGER NOT NULL DEFAULT 0;
ALTER TABLE mock_rules ADD COLUMN last_hit_at REAL;
```

Addon이 mock 매칭 시:
```sql
UPDATE mock_rules SET hit_count = hit_count + 1, last_hit_at = ? WHERE id = ?
```

### Core 신규 함수

- `core.query.delete_all_flows()` — 전체 flow 삭제
- `core.filter_parser.parse(text)` — `host:X status:4xx` 파싱 → dict
- `core.mock.suggest_glob(path)` — path에서 숫자/UUID를 `*`로 치환

### Data Flow

- **DB polling:** 0.5s 간격 `SELECT MAX(id) FROM flows` → 새 flow → table append. v0.2 `troxy tail` 구조 재사용.
- **Filter reactive:** FilterInput → debounce 200ms → core.query 호출 → FlowTable 갱신.
- **Detail body:** 초기 10KB만 로드, 스크롤 시 추가 fetch.
- **Mock dialog:** `m` → flow response prefill modal → 확인 → `core.mock.add()` → addon이 다음 요청부터 mock 반환.
- **My IP:** 앱 시작 시 탐지, 30s 간격 갱신. UDP dummy connect 방식 (default route 기준).

## Risks & Mitigation

- **Textual 성능 (10k+ flow)** — DataTable virtual scroll. 초기부터 fixture로 검증.
- **터미널 호환** — tmux/iTerm2/Terminal.app OK. Windows Terminal은 v0.3 out-of-scope.
- **Polling 부하** — v0.2 tail과 동일 패턴 (검증됨). 나중에 SQLite update hook 전환 가능.
- **병렬 충돌** — tui/ 파일 분리 철저. core/mock.py만 순서 제어.
- **copy.py 중앙화 실패** — W1 검수 시 반드시 확인. 톤 일관성 치명적.
