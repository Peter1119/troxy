**Goal:** mitmproxy read-only 유저와 Charles→mitm 전환자가 "troxy 써보니 편하다"고 느껴 자연스럽게 mock/intercept/Claude까지 쓰게 만들기.

## v0.2 범위 후보 (CEO/Eng/DX 리뷰 전)

### A. Adoption quick wins (P2/P3 중심)
1. **`troxy pick`** — interactive flow picker (native, no fzf dep). 화살표로 고르면 `flow <id>`로 전환.
2. **`troxy session`** — `session save/use/list`. 프로젝트별 DB를 이름으로 관리.
3. **`troxy install-cert`** — macOS keychain에 mitmproxy CA 자동 신뢰.
4. **Body truncation at write time** — `TROXY_MAX_BODY=1MB` env로 DB 비대화 방지.

### B. AI on-ramp (P1 → Claude 유저로)
5. **Claude hint injection** — `troxy flow 42` 하단에 "이 응답 해석이 필요하면: (in Claude) troxy_get_flow(42)" 한 줄. `troxy explain` 실패 시 "더 자세히 보려면 Claude에게 물어보세요" 힌트.
6. **`troxy mcp-tools`** — MCP에서 내보내는 툴 목록을 사람이 읽을 수 있게 출력. Claude 처음 쓰는 사람에게 "무엇이 되는지" 보여주기.

### C. Power-user delight (P2용)
7. **`troxy flows --jq '<expr>'`** — `--json | jq '.'` 패턴 단축.
8. **`troxy mock add --name`** — mock 이름으로 toggle/remove.
9. **Rule cache with version counter** — addon의 per-request DB 조회 제거.

### D. Safety / Hardening
10. **Body redaction at export** — `troxy flow 42 --redact authorization,cookie` 로 비밀번호 마스킹 후 curl 복사.
11. **Storage quota** — `troxy status`에 경고 (50MB 초과 시).

## Open questions (CEO/Eng/DX 리뷰로 풀 것)

1. **Scope**: A+B+C+D 다 할지, 절반만 할지? 다음 페르소나 만족과 복잡도 tradeoff.
2. **Interactive picker**: Python stdlib만으로 구현(ncurses) vs prompt_toolkit 신규 의존성?
3. **Session**: DB 경로 알리어스 저장 위치 — `~/.troxy/sessions.json` 새 파일 vs 기존 config와 합치기?
4. **install-cert**: macOS 전용이면 linux 유저는? WSL?
5. **Claude hint 과해지지 않나?** P2가 거부감 느끼면 역효과. `--no-hint` 기본값 or opt-out?
6. **Body truncation**: 데이터 손실 vs DB 크기. 기본값은 어떻게?
7. **MCP tool 리스트 CLI 노출**: 이걸로 Claude 미사용자가 MCP에 관심 생길까?

## Accepted v0.2 Scope (post CEO + Outside Voice)

1. `troxy pick` — curses interactive flow picker + **isatty guard** + `--last N` fallback (outside voice 반영)
2. ~~`troxy install-cert`~~ → **`troxy onboard`** (CA 생성 + trust + iOS/Android device proxy 안내) (outside voice 반영)
3. Claude hint injection — **MCP 등록된 경우에만 자동 표시** + `--no-hint` + `TROXY_HINTS=0/1` (outside voice 반영)
4. Named mock rules — `--name`, name으로 toggle/remove
5. `troxy session` — save/use/list (~/.troxy/sessions.json, 0600)
6. Body truncation — `TROXY_MAX_BODY` default 1MB (**new writes only**, historical flows unchanged)
7. `troxy mcp-tools` — MCP 도구 목록 + **example Korean prompts per tool** (outside voice 반영)
8. **`troxy explain` 강화 (outside voice가 요청한 8번째)** — Retry-After parse, Content-Type mismatch 감지, Cache-Control 진단 + 13개 golden-set tests
